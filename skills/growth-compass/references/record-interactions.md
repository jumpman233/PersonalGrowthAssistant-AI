# 记录交互规则

## 删除确认

删除记录是危险操作，必须二次确认。

当前项目使用本地可复用弹窗：

```text
app/components/common/ConfirmDialog.vue
```

弹窗要求：

- 显示遮罩。
- 非 pending 时 Esc 可关闭。
- 非 pending 时点击遮罩可关闭。
- pending 时确认按钮禁用。
- 删除失败时显示温和错误提示。
- 打开弹窗时不要隐藏页面滚动条；隐藏滚动条会导致横向布局跳动。

推荐文案：

```text
删除这条记录？
删除后它不会再出现在列表和统计里。这个操作不能撤销。
再想想
删除记录
```

除非恢复功能已经实现，不要承诺可以恢复。

## 详情页删除

- 详情页可以提供明确删除入口。
- 点击删除后打开确认弹窗。
- 确认后调用 DELETE 接口。
- 成功后跳转 `/records`。
- 失败时保留当前页面内容，并在弹窗里显示温和错误提示。

## 列表页删除

- 每张记录卡右下角提供弱化的垃圾桶按钮。
- 垃圾桶按钮使用 absolute 定位，不要撑大卡片边距。
- 点击垃圾桶必须 stop/prevent，不能同时跳转详情。
- 删除成功后只从当前前端数组移除，不重新拉取列表。
- 删除失败时保留列表，并在弹窗里显示温和错误提示。
- 删除按钮不能抢走记录阅读本身的注意力。

## Dashboard 最近记录

Dashboard 最近记录不添加删除按钮。

Dashboard 是概览页，主要动作是查看详情。危险操作不应该打断首页的安静感。

## RecordCard hover

记录列表卡片和 Dashboard 最近记录卡片需要统一 hover 感受。

推荐 hover：

```css
transform: translateY(-2px) scale(1.003);
transition: all 160ms ease-out;
```

使用：

- 稍微加深的暖色、奶油色或浅橙背景。
- 更清楚的低饱和橙色边框。
- 略增强的阴影。
- 很轻的上浮。
- 可选的轻微图标放大。

避免：

- hover 时向下移动。
- 放大幅度太大导致列表抖动。
- 强红或高饱和背景。
