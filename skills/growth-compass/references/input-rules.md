# 输入规则

这些规则适用于 `RecordForm`，包括 `/records/new` 和未来的 `/records/[id]/edit`。

## 校验规则

标题：

- 必填。
- 保存前 trim。
- 最多 80 个字。

内容：

- 必填。
- 保存前 trim。
- 最多 2000 个字。

评分：

- moodScore、constructivenessScore、energyCostScore 必须是整数。
- 范围是 0-5。
- energyCostScore 默认值是 2。

标签：

- 保存前 trim。
- 空标签不保存。
- 同一条记录内去重。
- 每条记录最多 12 个标签。
- 单个标签最多 20 个字。
- AI 推荐标签和手动输入标签都遵守同一规则。
- 推荐阶段不写入 Tag 表；只有保存记录时才根据最终 tags 创建或复用 Tag。

发生时间：

- 必填。
- 必须是有效时间。
- 新建记录默认当前时间。
- 编辑记录展示原记录发生时间。

## 错误交互

- 使用字段级错误。
- 提交失败时，平滑滚动到第一个出错字段。
- 错误提示展示在对应字段下方。
- 标题和内容需要监听输入事件。
- 当标题或内容因为“为空”触发错误后，只要用户输入的值 `trim()` 后不为空，就立即清除对应错误提示。
- 输入时不要误清除其他类型错误；例如标题超长、内容超长仍由提交校验和字数提示处理。
- 错误态使用低饱和颜色。
- 不依赖底部全局错误作为主要反馈。
- 保存按钮只在 pending 时禁用。
- 即使实时检测到超限，也允许点击保存，让校验流程负责滚动和说明原因。

## UI 提示

- 标题显示 `当前字数 / 80`。
- 内容显示 `当前字数 / 2000`。
- 标签显示已选数量和单个标签最大字数。
- 超限提示变成低饱和错误色。
- textarea 不允许拖拽调整大小。
