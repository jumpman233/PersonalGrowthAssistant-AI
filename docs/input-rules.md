# Input Rules

Growth Compass 的输入规则以“低压力记录”和“数据库写入稳定”为优先级。前端需要给出温和提示，后端必须做同等校验兜底。

## JournalRecord

- 标题必填，保存前 trim。
- 标题最多 80 个字。
- 内容必填，保存前 trim。
- 内容最多 2000 个字。
- 分类必填，必须是系统支持的 RecordCategory。
- 心情评分、真实建设感评分、内耗程度评分必须是整数，范围为 0-5。
- 内耗程度评分默认值为 2。
- 发生时间必填，必须是有效时间。

## Tags

- 标签保存前 trim。
- 空标签不保存。
- 同一条记录内重复标签去重。
- 每条记录最多选择 12 个标签。
- 单个标签最多 20 个字。
- 推荐标签或手动输入标签都必须遵守同一规则。
- 推荐阶段不直接写入 Tag 表；记录保存时再按最终 tags 创建或复用 Tag。

## UI Hints

- 内容输入区展示 `当前字数 / 最大字数`。
- 标题输入区展示 `当前字数 / 最大字数`。
- 标签输入区展示 `已选数量 / 最大数量` 和单个标签最大字数。
- 超出限制时，提示文案使用温和语气，不使用强压迫感措辞。
